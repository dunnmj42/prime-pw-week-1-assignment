# 5 - What output would you expect when you run the command 'ls' in an empty folder? What about a folder containing files?

ls in an empty directory would output nothing and return you to prompt, ls (with no arguments) in a directory containing files will list the non-hidden files in that directory

# 8 - Why would you run the command 'git status'?

to make sure that git "sees" all changes you have made to files since the last commit, which of those changes have been staged and which ones haven't, and to see any files that aren't being tracked by git

# 9 - What does the command 'git add .' do?

'git add .' adds any changed files in the current working directory to the 'staging area' to prepare them to be committed

# 10 - What is the command to push your committed changes to your repo on GitHub?

'git push <args> <repo> <branch>' ex: 'git push -u origin master'
